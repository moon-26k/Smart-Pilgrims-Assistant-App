# 🤝 Contributing Guidelines

First of all, thank you for considering contributing to the **Smart Pilgrims Assistant App**! It's people like you that make open-source AWESOME.

## 🚀 How to Contribute

### 1. Fork and Clone
1. **Fork** this repository by clicking the `Fork` button at the top right of this page.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Smart-Pilgrims-Assistant-App__Krish.git
   ```

### 2. Set Up the Project locally
Follow our [INSTALLATION.md](INSTALLATION.md) for detailed steps to get the app running locally.

### 3. Create a Branch
Use a clear and descriptive branch name:
```bash
# Good convention -> <type>/<short-description>
git checkout -b feature/sos-button
git checkout -b bugfix/login-crash
git checkout -b docs/readme-update
```

### 4. Make Changes & Commit
Write clear, concise commit messages. We prefer conventional commits:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation updates
- `style:` for formatting, missing semi colons, etc.

**Example**: 
```bash
git commit -m "feat: add user tracking on map"
```

### 5. Push Changes
```bash
git push origin feature/sos-button
```

### 6. Create a Pull Request (PR)
- Go to the original repository.
- Navigate to the **Pull Requests** tab.
- Click **New Pull Request** and compare across forks.
- Fill out the provided **PR Template** thoroughly.
- Link the PR to the relevant Issue (e.g., `Closes #12`).

## 🧠 Coding Standards
- Follow consistent indentation (Prettier/ESLint rules for frontend/backend).
- Always use meaningful variable names in CamelCase (JS) or snake_case (Python).
- Document new functions and complex logic.

## ⚖️ Code of Conduct
Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.
