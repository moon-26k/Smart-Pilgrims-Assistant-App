# Security Policy

## Supported Versions

Currently, the `main` branch of this repository implies the active development version. We actively support and patch security issues on the main release.

## Reporting a Vulnerability

If you discover a security vulnerability, please do NOT open a public issue. We appreciate your efforts to securely and responsibly disclose the issue to us.

**How to report:**
1. Send an email to the project maintainers directly or use GitHub's private vulnerability reporting feature (if enabled). 
2. Include the following details in your report:
   - A description of the vulnerability.
   - Steps to reproduce it.
   - Any possible solutions or mitigating factors.

We will review the report and respond within 48 hours to acknowledge it. After investigating, if the vulnerability is valid, we will release a patch as quickly as possible.

We appreciate your responsible disclosure and efforts to keep our community safe!
